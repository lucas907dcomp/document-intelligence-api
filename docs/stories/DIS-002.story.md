## User Story

> Como usuário, quero uma interface web de alto impacto para fazer upload de PDFs e conversar com seu conteúdo via linguagem natural, com feedback visual em tempo real do processamento.

---

## Contexto

O **Document Intelligence API** (DIS-001) está completo e rodando em `localhost:8080`. Esta story cobre a criação do frontend React/Vite/Tailwind que consome os 3 endpoints da API. O design segue o sistema "Vibe Pointer" — visual preto absoluto com acentos em gradiente verde menta (#20EFA4) / ciano (#00FFFF).

**API Base:** `http://localhost:8080`

| Método | Endpoint | Uso |
|--------|----------|-----|
| POST | `/documents` | Upload do PDF (multipart) |
| GET | `/documents/{id}/status` | Polling de status |
| POST | `/documents/{id}/query` | Chat com linguagem natural |

## Fluxo UX Completo

```
[Tela Inicial]
  ├── HeroSection (título + subtítulo animados)
  └── InteractionPanel
        └── UploadPanel
              └── DropZone: "Drop your PDF here or click to browse"
                      ↓ (drag / clique)
              [Validação inline: só PDF, máx 10MB]
                      ↓ (válido)
              [Barra de progresso de upload]
                      ↓ (202 Accepted → documentId)
[Estado: PROCESSING]
  └── ProcessingStatus
        └── Badge "Processing…" + spinner + nome do arquivo
        [Polling a cada 2s...]
                      ↓ (status = READY)
[Estado: READY]
  └── ChatInterface
        ├── (lista de mensagens vazia)
        ├── ChatInput: "Ask anything about your document…"
        │       ↓ (usuário envia pergunta)
        │   [Loading spinner no botão]
        │       ↓ (200 OK)
        ├── MessageBubble (user): pergunta
        ├── MessageBubble (AI): resposta
        │       └── SourceChunks [accordion]: trecho 1, trecho 2...
        └── Botão "Upload New Document" → reset para idle
```

## Notas para o Dev

1. **Proxy Vite é suficiente em dev** — não mexer no `CorsConfig.java` do backend ainda.
2. **Polling para sozinho** — o `refetchInterval` retorna `false` quando status ≠ `PROCESSING`. Não usar `setInterval` manual.
3. **Gradiente de borda** — implementar via wrapper `<GradientBorder>` (div com `p-px` e background gradiente, envolvendo o painel filho com `rounded-2xl bg-zinc-950`). Pseudo-elementos funcionam mas ficam mais complexos.
4. **Scroll automático no chat** — usar `useEffect` + `ref` no último elemento da lista de mensagens.
5. **shadcn init** — selecionar "Default" style e "zinc" como base color durante `npx shadcn@latest init`. Ajustar manualmente para o dark theme Vibe Pointer após.
6. **TypeScript strict** — manter `strict: true` no tsconfig. Os tipos em `api.types.ts` devem cobrir todos os campos da resposta da API.
